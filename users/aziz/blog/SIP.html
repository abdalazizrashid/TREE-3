System Integrity Protection

SIP is another layer of security used in MacOS to prevent the unconstrained powers of the root user. This means with SIP the root user can no longer perform certain actions such as modifying special file or add kernel extension modules. Those files will only be modifible by Apple itself and techincally should be considered Apple ownership.

Files under /bin /sbin/usr /Sytem are protected while /usr/local is the only directory in /usr is availble for developers. The directories annotated by asterisks are an exception and are not protected by SIP. We can see which files are protected by inspecting the output of ls -O note that ls from GNU coreutils won't work as the -O is not complaint with IEEE Std 1003.1-2008 (“POSIX.1”), we need the bsd variant which is shipped by MacOS available at /bin/ls.

Files annotated with restricted are protected by the SIP. For a list of the flags meanings chflags(1) - change file flags. An excerpt of the manual page is below:

The flags are specified as an octal number or a comma separated list of keywords. The following keywords are currently defined:

arch, archived set the archived flag (super-user only)

nodump set the nodump flag (owner or super-user only)

opaque set the opaque flag (owner or super-user only) [Directory is opaque when viewed through a union mount]

sappnd, sappend set the system append-only flag (super-user only)

schg, schange, simmutable set the system immutable flag (super-user only)

uappnd, uappend set the user append-only flag (owner or super-user only)

uchg, uchange, uimmutable set the user immutable flag (owner or super-user only)

hidden set the hidden flag [Hide item from GUI]

Modifying the SIP system

The SIP protection can't be disbled from the user space with a typical login, it can be change from the MacOS recovery environment, as the SIP configuration is stored in the NVRAM. The tool used to manage SIP is /usr/bin/csrutil this tool have special entitlements allowing it to modify the NVRAM, we can inspect it running: 

codesign -d --entitlements - /usr/bin/csrutil

In recovery mode we can enable and disable the SIP system 

/usr/bin/csrutil

Modify the System Integrity Protection configuration.
Available commands:

    clear
        Clear the existing configuration.
    disable
        Disable the protection of the OS installation. Only available in Recovery OS.
    enable
        Enable the protection of the OS installation. Only available in Recovery OS.
    status
        In Recovery OS, displays the configuration for each OS installation.
        In macOS, displays the configuration of the running OS.

    authenticated-root
        status
            Show the current authenticated root setting.
        disable
            Allow booting from non-sealed system snapshots. Only available in Recovery OS.
        enable
            Only allow booting from sealed system snapshots. Only available in Recovery OS.

Enable and disbale SIP restrictions

  • dtrace csrutil enable --without dtrace
  • restrictions or writing to NVRAM csrutil enable --without nvram
  • debugging restrictions csrutil enable --without debug

Finally to see the current status run csrutil status.
